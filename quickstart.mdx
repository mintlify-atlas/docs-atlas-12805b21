---
title: "Quickstart"
description: "Register, connect your first API, and download a generated Django site in under 5 minutes."
---

<Steps>
  <Step title="Register an account">
    Go to `/registro` and fill in your username, email, and password. After submitting, WebBuilder automatically logs you in and redirects you to the home page.

    <Note>
      n8n sends a welcome email to the address you registered with. If you do not receive it, check that `N8N_WEBHOOK_REGISTRO` points to a running n8n instance and that your n8n workflow is active.
    </Note>
  </Step>

  <Step title="Open the assistant">
    Navigate to `/asistente`. This is the four-step generation wizard where all site creation happens.
  </Step>

  <Step title="Enter a public API URL">
    Paste any URL that returns JSON or XML. WebBuilder fetches the response, detects the format automatically, and safely parses it.

    Some good examples to try:

    ```text
    https://jsonplaceholder.typicode.com/posts
    https://api.publicapis.org/entries
    ```

    You can also upload a local `.json` or `.xml` file using the file input on the same form.
  </Step>

  <Step title="Choose an LLM model">
    Select a model from the catalog shown on the analysis page. The catalog includes models from OpenRouter and Groq. If you have configured a personal LLM in your profile (`/perfil`), that model is used instead of the server default.
  </Step>

  <Step title="Review the AI plan">
    The LLM analyses the dataset and returns a structured plan that includes:

    - The recommended site type (blog, catalogue, portfolio, dashboard, and so on)
    - The relevant fields extracted from the dataset, with human-readable labels
    - A normalised preview of the data

    Accept the plan to proceed, or enter a custom natural-language prompt and click **Regenerate** to have the LLM revise it. You can iterate as many times as you like before moving on.

    <Note>
      Every regeneration is a separate LLM call. Keep your prompt specific — for example, "focus only on the `title` and `body` fields and present them as a news site" — to get the best results quickly.
    </Note>
  </Step>

  <Step title="Generate the project">
    Click **Generate**. WebBuilder sends the approved plan to the LLM, which produces a complete Django project including:

    - `models.py`, `views.py`, and `urls.py` tailored to your dataset
    - HTML templates styled with Tailwind CSS
    - A `load_data` management command that fetches and seeds the database from the original API
    - A `Dockerfile` and `entrypoint.sh` for containerised deployment

    Generation typically takes between 20 and 60 seconds depending on the model and dataset size.
  </Step>

  <Step title="Download the ZIP and run the site locally">
    Click **Download ZIP** to save the generated project. Then run it locally:

    ```bash
    unzip mysite.zip
    cd mysite
    pip install -r requirements.txt
    python manage.py migrate
    python manage.py load_data
    python manage.py runserver
    ```

    Open `http://127.0.0.1:8000` to see the generated site.

    <Tip>
      If the generated site includes user accounts, run `python manage.py seed_users` after `load_data` to create the default user fixtures.
    </Tip>
  </Step>
</Steps>
